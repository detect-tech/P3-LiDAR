01 — The Problem

NEET's current process does not effectively utilize LiDAR data

Planning

  1. 1Utilize Routify to generate an 80% solution to the planned route
  2. 2Review the route with SMEs to identify additional constraints
  3. 3Revise the route to account for those constraints
Key takeaway: LiDAR data is not consistently used in planning.

Execution

  1. 1Begin acquiring land and permits along the preplanned route
  2. 2Reroute based on the land actually acquired
  3. 3Purchase LiDAR as required
Key takeaway: LiDAR is a potential critical-path item if late re-routing occurs.
02 — The Solution

Integrated LiDAR platform synthesizes public and internal data to improve routing and de-risk execution

1

Import the route

User inputs the initial route directly from Routify.

→
2

Evaluate against existing data

Tool evaluates the route against the existing database, identifying data gaps and potential hazards.

→
3

Return a report

Tool returns a report listing:

  • Data gaps
  • Areas with insufficient data resolution
  • Potential hazards for users to confirm
Key takeaway: the tool provides a unified system to store and rapidly query LiDAR data.
03 — The Value

Tool improves bid quality and de-risks project execution

Planning benefits

  • Reduces the time required of SMEs to review initial routes
  • Identifies hazards based on additional data, at no cost
  • Improves design and bid quality — potentially increasing NEET's win rate

Execution benefits

  • Eliminates potential spend on duplicate data
  • Provides early visibility into data gaps, enabling pre-planning
  • Provides early data to improve E&C designs
